API

You'll need to access the Wyam interfaces when writing a module or when working with configuration files. The interface definitions reside in the Wyam.Abstractions library, but you'll only need to worry about referencing it if you're developing modules or embedding Wyam.

IDocument

The primary object in Wyam that contains content and metadata for each item as it propagates through the pipeline.

IDocumentCollection

Allows you to get sequences of documents from the current and previous pipelines.

IExecutionCache

Used by modules to cache data for faster build times.

IExecutionContext

Contains all the information about the running execution and pipeline.

IIndentedTraceEvent

Provides an easy way to indent trace messages.

IMetadata

Contains the metadata for a document.

IMetadata<T>

Contains the metadata for a document converted to a specified type.

IModule

Modules must implement this interface.

IPipeline

Contains information about a pipeline.

IPipelineCollection

Contains a collection of all configured pipelines.

IReadOnlyPipeline

Contains read-only information about a pipeline.

ITrace

An interface for outputting trace and log messages.
